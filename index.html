Layout studies

A collection of sketches meant to study possible layouts and layout combinations on the web.

Fixed amount of items

We often have complete control over the exact amount of items on a page, or on a section of a page. It is interesting to research how different amounts of items react to one another. It's also interesting to research if different types of content react differently in the same situations.

Single item

There are many ways to display one single item. There are many different types of items, of course, but the way we show them can vary considerably, depending on many factors, like screen size for instance.

Examples with text

What are the possible layouts, or combinations of layouts for one single text item? A text item can be any text, long or short, with no other content to share the layout with. An article, for instance.

  • A single, flexible column

    The most basic possible layout. The archetypical web layout of the ninetees. One flexible text item without a maximum width.
  • A single, flexible column with a maximum with

    A very basic layout. The fluid web layout sometimes seen in the early years of 2000. One flexible text item with a maximum width in order to keep things readable.
  • Flexible, single or multi-column layout with max-width

    On bigger screens, it uses a simple three column grid to make the first heading and the first paragraph stand out. This kind of layout is especially useful for longer articles. Other elements can span across all three columns as well, if needed. It's a rather classic layout, sometimes seen in books and magazines.
  • Flexible, single or multi-column layout with max-width

    The same as the previous example, but with an extra column for wider viewports, in which things like marginal notes can be shown.
  • Single, flexible column with changing percentages, with a rather large max width

    This is a simple looking one column layout. The margins to the left and right change a few times, at different viewport sizes. The width of the measure grows from 91% to 60% with a max-width of 44.5em. But there's more. An interesting thing about this layout is that the first paragraph is 7% wider that the body copy, and the heading is even wider: 15%
  • Single, flexible small column with two fixed widths

    This is a rather simple single column layout. It starts out with a width of 90% on smaller screens, with a max width of 27em. It then stops for a while and collects a lot of white-space to the left and right. Once there is room enough, the width of the measure is set to 41.375em, and the font size is increased.
  • Flexible, single or multi-column layout with some max-widths and some parts spanning multiple columns

    This is a set of layouts that gets more complex as the viewport grows. At first everything is layed out over one single column, but as soon as the measure allows for an extra column, it is added to the left. This can be repeated.

Two items

There are many situations that two different items have to be displayed in relation to each other on a page, or in a section of a page. In this section, a study is made of how these situations can be layed out.

Example with text

How can two individual, but somehow related text items be layed out. There are many different answers to this question. Here are some of the options.

  • One on top of the other

    This is probably the most simple way to display two items on a page. The one always on top of the other. No CSS is needed to position the elements, this is how things flow naturally. So simple, you might actually forget this is a valid option.

  • One next to the other

    On a small screen, the one item is always on top of the other. On bigger screens, when the minimum measure allows it, the items are shown next to each other. Since two text-items will probably have different hights, we can decide things about alignment. In this version the items are vertically aligned relatively to each other's top.
  • One next to the other with vertical-align: middle

    The same as the previous version. But in this version the items are vertically aligned relatively to each other's middle, which sometimes works nicely for items with uncontrolably long or short headings.
  • One next to the other with vertical-align: bottom

    The same as the previous version. But in this version the items are vertically aligned relatively to each other's bottom, which sometimes works nicely.
  • One on top of the other, the first one a bit bigger

    The first item in a list is often more important. There are different ways to emphasize this, one of them is using a larger font-size. On small screens, the first item will just have a larger font-size, on larger screens the first item will also be wider, because the width of the measure is related to the font-size
  • The other on top of the one, the second one a bit bigger

    The first item in a list is often more important, but sometimes you don't want to show it as the first item. The first item should probably be bigger, but in this case not first. On small screens, the first item will just have a larger font-size, on larger screens the first item will also be wider, because the width of the measure is related to the font-size. Works only in flexbox enabled browsers
  • One on top of the other, the second one a bit bigger, with reversed order for wider viewports

    Similar to the previous version. In this version, on small screens, the first item is visually displayed as the first item. On bigger screens, it's shown as the second one. Works only in flexbox enabled browsers
  • One on top of the other, the second one a bit bigger, with reversed order dependent on aspect ratio

    Similar to the previous version. In this version, on small screens and on big screens with viewports with an aspect-ration less than 1/1, the first item (this one) is shown as the first item. On bigger screens with an aspect-ratio bigger than 1/1, it's shown as the second one. Works only in flexbox enabled browsers
  • One on top of the other, the second one a bit bigger, with reversed order dependent on viewport height

    Similar to the previous version. In this version, on lower screens, the first item is shown as the first item. On higher viewports, it's shown as the second one. Works only in flexbox enabled browsers
  • The second item is more important

    By simply making the font-size of the second item bigger it gets more attention. Very simple, pretty effective. It might be less effective on smaller screens when the second item is not immediately visible.
  • The second item is more important

    Another variant where the second item is more important. This time, if the screen is wide enough, the two items are shown next to each other.

Three items

There are many situations that two different items have to be displayed in relation to each other on a page, or in a section of a page. In this section, a study is made of how these situations can be layed out.

Examples with text

How can three individual, but somehow related text items be layed out. There are many different answers to this question. Here are some of the options.

  • Three items, the one above the other

    The simplest way to display three items is to put one above the other. Works on all kinds of screens.
  • Three items, the one above the other on small screens, next to each other when it fits

    On wider screens we can show items next to each other. The hard part here is finding a logical breakpoint to switch from one to three columns without breaking the layout on any resolution.
  • Three items, the one above the other on small screens, next to each other when it fits

    On wider screens we can show items next to each other. The hard part here is finding a logical breakpoint to switch from one to three columns without breaking the layout on any resolution.
